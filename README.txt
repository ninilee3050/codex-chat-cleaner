Codex 채팅 삭제기

실행:
1. "Run Codex Chat Cleaner.bat" 파일을 더블클릭합니다.
2. 삭제할 채팅의 체크박스를 누릅니다.
3. "스마트 정리" 버튼을 누릅니다.
4. "남은 찌꺼기 확인"으로 이미 목록에서 사라진 세션 찌꺼기를 확인할 수 있습니다.
5. 생성 이미지는 상단의 "이미지 관리" 버튼을 눌러 같은 화면에서 관리합니다.

파일 역할:
- Run Codex Chat Cleaner.bat: 더블클릭 실행용 파일입니다. 가능하면 CMD 창 없이 실행합니다.
- codex_chat_cleaner.py: 실제 프로그램 본체입니다. 삭제하면 안 됩니다.
- README.txt: 이 설명 파일입니다.

목록:
- 일반 채팅 세션만 표시하고 검색합니다.
- Codex의 state DB, session_index.jsonl, sessions 폴더의 rollout 파일을 합쳐서 세션 목록을 찾습니다.
- "The following is..."로 시작하는 내부 검토 기록은 목록에서 숨깁니다.
- 출처가 guardian으로 표시되는 내부 검토 기록도 목록에서 숨깁니다.
- 수정일 기준 최신순으로 정렬됩니다.
- 출처/모델 열은 참고 정보입니다.
- "열기" 버튼은 해당 채팅 세션이 사용한 작업 폴더를 엽니다.

버튼:
- 새로고침: Codex 채팅 목록을 다시 읽습니다.
- 이미지 관리: 채팅 목록 영역을 생성 이미지 미리보기 목록으로 바꿉니다.
- 남은 찌꺼기 확인: threads 목록에는 없지만 로그/목표/인덱스/전역 상태에 남은 thread id를 읽기 전용으로 검사합니다.
- 용량 줄이기: 삭제 후 DB 파일 안에 남은 빈 공간을 회수합니다. Codex 앱이 DB를 사용 중이면 실패할 수 있습니다.
- 보이는 항목 모두 체크: 현재 검색 결과에 보이는 항목을 전부 체크합니다.
- 체크 해제: 모든 체크를 풉니다.
- 스마트 정리: 체크한 채팅과 관련 찌꺼기를 삭제하고, 이미 남아 있던 찌꺼기도 함께 정리합니다.
- 삭제 확인창은 프로그램 중앙에 열립니다.

보호:
- 각 채팅 행의 "보호" 체크를 켜면 해당 채팅을 삭제 대상에서 제외합니다.
- 보호 체크는 프로그램을 껐다 켜도 유지됩니다.
- 보호 체크를 끄면 다시 체크하고 삭제할 수 있습니다.
- 최근 10분 이내 수정된 세션은 실수 방지를 위해 자동보호로 표시됩니다.

스마트 정리 동작:
- 체크한 채팅이 있으면 체크한 채팅과 관련 찌꺼기를 삭제합니다.
- 체크한 채팅이 없으면 이미 남아 있던 찌꺼기만 정리합니다.
- 체크하지 않은 정상 채팅 세션은 자동 삭제하지 않습니다.
- "보호" 체크가 켜진 세션은 삭제하지 않습니다.
- [자동보호] 표시가 붙은 최근 세션은 삭제하지 않습니다.
- Codex 최근 목록 DB에서 해당 채팅을 제거합니다.
- 실제 세션 .jsonl 파일도 함께 삭제합니다.
- logs_*.sqlite의 해당 thread 로그도 함께 삭제합니다.
- goals_*.sqlite의 해당 thread goal도 함께 삭제합니다.
- memories_*.sqlite의 해당 thread memory output도 함께 삭제합니다.
- 해당 채팅 ID를 참조하는 내부 검토 기록도 자동으로 함께 삭제합니다.
- 해당 채팅이 쓰던 작업 폴더가 비어 있으면 함께 삭제합니다.
- 작업 폴더 안에 파일이 있으면 자동 삭제하지 않습니다.
- 관련 session_index.jsonl, .codex-global-state.json, .codex-global-state.json.bak 항목이 있으면 같이 정리합니다.
- 세션 파일 삭제 뒤 비어 있는 sessions 날짜 폴더도 정리합니다.
- Codex 버전에 따라 일부 DB 테이블이 없어도 실패하지 않고 건너뜁니다.
- 정리 후 DB 빈 공간이 충분히 클 때만 자동으로 용량 줄이기를 시도합니다.
- 용량 줄이기가 실패해도 삭제 정리는 완료된 것으로 봅니다. 나중에 Codex를 닫고 다시 시도할 수 있습니다.

남은 찌꺼기 정리:
- state DB와 session_index/rollout 파일로 확인되는 정상 세션은 남은 찌꺼기로 보지 않습니다.
- 정상 세션 목록에는 없지만 logs/goals/memories/session_index/global_state에 남은 thread id를 남은 찌꺼기로 봅니다.
- "남은 찌꺼기 확인"은 삭제하지 않고 남은 수량만 보여줍니다.
- "스마트 정리"는 확인창을 보여준 뒤 남은 찌꺼기를 함께 삭제합니다.
- 최근 활성 세션 또는 전역 상태에서 현재 projectless 세션으로 보이는 thread id는 기본 보호합니다.
- state DB에는 있지만 rollout 파일이 없는 세션은 검사 결과에 표시하지만 자동 삭제하지 않습니다.

이미지 관리:
- C:\Users\user\.codex\generated_images 안의 이미지 파일을 최신순으로 보여줍니다.
- 이미지 미리보기를 갤러리 형태로 보고 체크한 뒤 "선택 이미지 삭제"로 지울 수 있습니다.
- 갤러리는 최소 4칸으로 보이고, 프로그램 창을 넓히면 칸 수가 자동으로 늘어납니다.
- 각 이미지의 "크게 보기" 버튼을 누르면 사용자가 선택한 이미지만 크게 보여줍니다.
- 크게 보기 이미지는 화면의 약 65% 안에서 원본 비율을 유지합니다.
- 크게 보기 창은 삭제기 중앙에 하나만 열리고, 이미지를 클릭하거나 Esc를 누르면 닫힙니다.
- 이미지가 많으면 같은 목록 안에서 스크롤됩니다.
- "채팅 세션"과 "이미지 관리" 버튼은 항상 같은 위치에 고정됩니다.
- 왼쪽 통계 문구도 현재 화면에 맞춰 "전체 세션" 또는 "전체 이미지"로 바뀝니다.
- 선택한 이미지가 삭제되어 비어버린 하위 폴더는 함께 삭제합니다.
- generated_images 폴더 바깥의 파일은 삭제하지 않습니다.

주의:
- 중요한 채팅은 먼저 Codex의 보관 기능을 사용하세요.
- 삭제기에는 별도 백업 기능이 없습니다.
- 작업 중인 Codex 세션은 기본 보호되지만, 대량 삭제 전에는 "남은 찌꺼기 확인" 결과를 먼저 확인하세요.
- "용량 줄이기"는 직접 누르지 않아도 됩니다. 스마트 정리가 필요할 때 자동으로 시도합니다.
- 사용 중인 Codex 창이 목록을 오래 들고 있을 수 있으니, 삭제 후 Codex를 다시 열면 더 깔끔합니다.
